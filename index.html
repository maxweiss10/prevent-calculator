How it works: Paste your Epic PREVENT dot phrase, a labs/results
      view, or a note — everything is parsed in your browser (no data leaves the page) and pre-fills
      the editable form above. It reads the values with deterministic pattern rules (not AI), so it:

    
    
      
  • normalizes Unicode (smart quotes, non-breaking spaces, en/em-dashes) so copy-paste artifacts don’t break matching;
    • 
      
  • scrapes labs & vitals from result tables, @LASTBP(n)@ reading lists,
        comma lists, and prose — taking the systolic of the most recent BP and ignoring dates/reference ranges;
    • 
      
  • detects threshold values (like “eGFR >60”) and warns you that the parsed number is a
        reporting limit, not an exact measurement — enter the actual value if known;
    • 
      
  • computes eGFR from serum creatinine (CKD-EPI 2021) when only creatinine is shown;
    • 
      
  • detects diabetes, statin, antihypertensive, and smoking from problem-list / medication /
        social-history text: a listed drug or diagnosis ⇒ Yes; when the source is present but the item
        isn’t in it (no diabetes on the problem list, “No current … medications”) ⇒ No, flagged to verify.
        Guards exclude pre-diabetes, family history (including multi-line family-history sections), allergies,
        discontinued meds, and negated phrases (“not a current smoker”). If there’s no source to
        rule an item out, it’s left blank for you to enter;
    • 
      
  • treats “not on file”, “N/A”, “unknown”, and similar phrases as missing data, not as a clinical No;
    • 
      
  • validates cross-field plausibility (e.g. total cholesterol ≤ HDL suggests swapped values);
    • 
      
  • auto-selects the model (base, or +HbA1c / +UACR / +SDI / full) from whichever optional values are present.
    • 
      
  • runs a second, independent parser (a different algorithm — number/unit harvesting instead of
        label-anchoring) and cross-checks the two. When they disagree on a value — including when a stated
        eGFR contradicts the serum creatinine — the field is flagged in red as a
        likely parsing error to double-check. The form keeps the primary parser's value.
    • 
      
  • shows its work (“What the parser read”): the pasted text is echoed back with every value it used
        highlighted and color-coded to a field — so a number it missed stands out (a cholesterol in mmol/L, a
        lab it couldn’t label) and gets flagged “not captured — check.”
    • 
      
  • quantifies what’s at stake: for a disagreement or an auto-detected flag, it recomputes 10-year ASCVD
        both ways (e.g. “eGFR 92 → 13.7% vs eGFR 24 → 21.6%”), so you know whether an uncertain value actually
        changes management or is safe to move past.
    • 
    

    
Values read from the paste show a green dot; auto-detected ones show an
      amber dot and a “verify” note. Nothing has to be perfect in Epic — every field
      is editable, so confirm the values before reading the result.

    
Model: AHA PREVENT equations (base + optional CKM/SDI). Coefficients from the
      Supplemental Appendix of Khan SS, Matsushita K, Sang Y, et al.
      “Development and Validation of the American Heart Association’s PREVENT Equations.”
      Circulation. 2024;149(6):430–449. This tool’s engine was ported from and verified
      bit-for-bit against the CRAN preventr
      package (v0.11.0) across 1,000 test cases (all 5 outcomes, both sexes, both horizons); eGFR-from-creatinine
      matches the same package. Source & verification.

    
Scope: 10-year risk is validated for ages 30–79; 30-year risk for ages 30–59.
      ASCVD risk categories (low/borderline/intermediate/high) follow the 2018/2019 ACC/AHA thresholds,
      which were derived with the Pooled Cohort Equations — apply clinical judgment.
